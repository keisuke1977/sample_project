# プロダクト要件

## 1. プロダクト概要

### プロダクト形態
- **従業員向け：** Webアプリ（スマートフォン・PCブラウザ対応）
- **企業・人事向け：** Webダッシュボード（PC・スマートフォンブラウザ対応）

### 対応プラットフォーム
- Webブラウザ：Chrome / Safari / Edge（従業員向けアプリ・管理ダッシュボード共通）
- スマートフォン・タブレット・PCすべてに対応（レスポンシブデザイン）

### 利用環境
- 従業員：通勤中・昼休み・就寝前など、スマホブラウザを使えるタイミング
- 人事担当（PC）：月次・週次でのデスクワーク中（詳細分析・レポート出力）
- 人事担当（スマートフォン）：外出中・移動中のサマリー確認・アラート通知の受け取り

## 2. コアバリュー

### このプロダクトの核となる価値
「忙しくても続けられる毎日1分のチェックが、女性の体の変化への気づきを生み、必要な時に専門家につながれる」

### Aha Moment
- 従業員：「先週と体調の傾向が違う、月経前だからか」と自分の体のパターンに気づいた瞬間
- 従業員：チャットで気軽に専門家に相談できて「受診すべきか」がわかった瞬間
- 人事：「うちの女性社員の体調傾向がこんなにわかるのか」とダッシュボードを見た瞬間

## 3. 主要機能

### 必須機能（Must Have）
- 毎日の体調チェック（タップ式、1分以内）
- チェック結果に基づくパーソナライズフィードバック（「今日の気づき」）
- 女性ライフステージ別コンテンツ（月経・PMS・更年期・妊活）
- 看護師・助産師へのチャット相談
- 産婦人科医とのオンライン相談（テキスト＋ビデオ）
- 人事向け匿名集計ダッシュボード（部署別・年代別）
- 健康経営レポート自動生成

### 推奨機能（Should Have）
- 体調履歴グラフ（月次・週次で自分の傾向を確認）
- 月経周期トラッキング
- 休職・離職リスクアラート（匿名集計ベース）
- 健康経営優良法人認定申請データ出力
- プッシュ通知（チェックインリマインダー・相談返信通知）

### あれば嬉しい機能（Nice to Have）
- 産業医との連携機能
- 他の健康データとの連携（歩数・睡眠：Apple Health / Google Fit）
- 社内の健康イベント・セミナー案内
- 同僚との匿名コミュニティ（将来検討）
- 多言語対応

## 4. 非機能要件

### プライバシー・セキュリティ（最重要）
- 従業員の個人体調データは企業側から絶対に特定できない設計
- データの匿名化・集計処理は5人以上のグループ単位（個人特定防止）
- 医療情報としての適切なデータ暗号化・管理
- プライバシーポリシーの明示と同意フロー

### パフォーマンス
- チェックイン画面の表示：1秒以内
- 相談メッセージの送信：即時（非同期）
- ダッシュボードのデータ反映：リアルタイム〜24時間以内

### スケーラビリティ
- MVP：数社・数百人規模
- 1年後：数十社・数万人規模を想定した設計

### 医療規制対応
- 医師法・薬機法の範囲内での「相談」に留める（診断・処方は行わない）
- 「受診を促す」ことを明示し、医療行為とならない設計
- 専門家の監修のもとコンテンツを制作・管理
