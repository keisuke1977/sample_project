# MVP範囲

## MVP定義

### 目的
パイロット導入企業（1〜3社）での実証を通じて、以下を検証する：
1. 従業員が毎日チェックインを継続するか（習慣化できるか）
2. 専門家相談が実際に使われるか
3. 人事担当者がダッシュボードを活用し、継続契約の意向を持つか

### 対象ユーザー
- 従業員：大手企業の女性従業員（パイロット企業の希望する部署・チーム単位）
- 企業：人事部・健康経営推進担当

### 成功の定義
| 指標 | 目標値 | 計測タイミング |
|------|--------|----------------|
| 月次アクティブ率 | 登録ユーザーの50%以上が月4回以上チェックイン | パイロット開始1ヶ月後 |
| 専門家相談利用率 | アクティブユーザーの20%以上が1ヶ月以内に相談利用 | パイロット開始2ヶ月後 |
| 企業継続意向 | パイロット終了後に正式契約意向ありが80%以上 | パイロット終了時 |
| NPS（従業員） | 40以上 | パイロット終了時アンケート |

## スコープ

### MVP に含める ✅

**従業員アプリ**

| 機能 | 理由 |
|------|------|
| 招待コード登録・アカウント作成 | 利用開始の必須フロー |
| プロフィール設定（年代・ライフステージ） | パーソナライズの基礎 |
| プライバシー同意フロー | 信頼性確保のため必須 |
| 毎日の体調チェック（5〜7問） | コア体験 |
| 「今日の気づき」フィードバック | 継続動機の核 |
| チェックインリマインダー（Web Push通知 / メール通知） | 継続率に直結 |
| コンテンツ閲覧（記事形式） | 「学び」ステップの提供 |
| チャット相談（看護師・助産師） | 差別化の核・相談仮説の検証 |
| オンライン相談予約・実施（産婦人科医） | 差別化の核 |

**企業ダッシュボード**

| 機能 | 理由 |
|------|------|
| 管理者アカウント・部署設定 | 導入に必須 |
| 従業員一括招待 | 導入に必須 |
| 全体・部署別体調傾向ダッシュボード | 企業への価値提供の核 |
| 月次レポートPDF出力 | 企業の購買理由 |
| 健康経営優良法人認定用データ出力 | 企業の購買理由 |

### MVP に含めない ❌

| 機能 | 理由 | 対応時期 |
|------|------|----------|
| 体調カレンダー・グラフ | MVP後の継続動機として実装 | Phase 2 |
| 月経周期トラッキング | 初期は簡易入力で代用 | Phase 2 |
| 短動画コンテンツ | 記事形式で代替可能 | Phase 2 |
| 休職・離職リスクアラート | データ蓄積後に実装 | Phase 2 |
| Apple Health / Google Fit連携 | Webアプリからの直接連携は困難なため対象外 | 将来検討（ネイティブアプリ化時） |
| 産業医連携 | 関係者が増え複雑になる | Phase 3 |
| コミュニティ機能 | モデレーション負荷が高い | 将来検討 |
| 多言語対応 | 国内大手企業向けはまず日本語 | 将来検討 |
| カスタムレポート | 月次PDFで代替可能 | Phase 2 |

## タイムライン

| フェーズ | 期間 | マイルストーン | 主な作業 |
|----------|------|----------------|----------|
| MVP開発 | 1〜3ヶ月目 | MVPリリース | 従業員向けWebアプリ（チェックイン・コンテンツ・相談）＋ダッシュボード基本機能の開発。専門家ネットワーク（産婦人科医3名・助産師5名）との契約完了 |
| パイロット | 3〜6ヶ月目 | 導入企業3社・登録ユーザー300名 | パイロット企業での試験運用。週次インタビュー・データ分析。MAU目標：登録者の50%以上 |
| 正式リリース | 6〜9ヶ月目 | 導入企業5社 | パイロット改善版リリース。営業活動本格化（健康経営イベント出展・代理店開拓）。Phase 2機能（体調グラフ・周期トラッキング）追加 |
| 成長期 | 9〜12ヶ月目 | 導入企業10社・ARR目標1億円 | 健保連携交渉。産業医連携機能のロードマップ策定。導入事例コンテンツ作成 |

## 制約事項
- 医療行為（診断・処方）には踏み込まない設計が必須
- 個人体調データの匿名化・企業側への非公開は絶対条件
- 専門家ネットワーク（産婦人科医・助産師・看護師）の確保がMVP前提条件

## リスクと対策

| リスク | 対策 |
|--------|------|
| 専門家の確保が難航する | 既存のオンライン医療プラットフォームとの提携を先行検討 |
| 従業員がチェックインを続けない | 通知設計・フィードバックのUXに注力。初月の継続率を重点KPIに設定 |
| 企業がプライバシーを懸念して導入しない | 匿名化の仕組みを契約前に明示。第三者審査・認定の取得を検討 |
| 医療規制への抵触 | 弁護士・医師監修のもと「相談」と「診療」の線引きを明文化 |
