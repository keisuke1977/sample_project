# UI/UX方針

## 1. 全体の方向性

### デザインテーマ
**「やさしい、プロフェッショナル」**
医療・健康という繊細なテーマを扱うため、清潔感・信頼感を保ちながらも、
「病院のような堅さ」ではなく、女性が毎日使いたくなる温かみのある雰囲気にする。

### キーワード
- **安心感** - 体の悩みを打ち明けられる、安全な場所
- **シンプル** - 忙しい中でも迷わず使える
- **温かみ** - 冷たくない、寄り添うトーン
- **信頼** - 医療専門家が関わっている安心感
- **女性らしさ（押しつけない）** - 華美でなく、自然な上品さ

### 参考サービス [仮]

| サービス名 | 参考にする点 |
|------------|--------------|
| Notion | シンプルで整理されたレイアウト、余白の使い方 |
| Calm / Headspace | 体験の温かみ、通知・フィードバックのトーン |
| Flo（月経管理アプリ） | 女性健康アプリの情報設計、グラフ表示 |
| Stripe | 清潔感のあるプロフェッショナルなUI |

### タイポグラフィ方針

| 役割 | フォント | ウェイト | サイズ目安 |
|------|----------|----------|------------|
| 見出し（H1） | Noto Sans JP | Bold (700) | 24px |
| 見出し（H2） | Noto Sans JP | SemiBold (600) | 20px |
| 見出し（H3） | Noto Sans JP | SemiBold (600) | 17px |
| 本文 | Noto Sans JP | Regular (400) | 15px |
| キャプション | Noto Sans JP | Regular (400) | 12px |
| ボタン | Noto Sans JP | Medium (500) | 15px |

- **行間（line-height）：** 本文 1.75、見出し 1.4
- **フォールバック：** `"Hiragino Sans", "Yu Gothic", sans-serif`
- **英数字：** Inter（Google Fonts）を Noto Sans JP と組み合わせ
- **雰囲気：** 読みやすく、医療文書のような硬さのないフレンドリーな印象

### アイコン・イラスト
- **アイコンセット：** Lucide Icons（オープンソース）またはPhosphor Icons / ライン系
- **スタイル：** 線幅 1.5px、角丸あり。塗りつぶし系は避ける
- **イラスト：** 使用する場合はシンプルなフラットスタイル（Blush Design / unDraw 参考）
- **人物イラスト：** 年代・体型の多様性を意識。特定のステレオタイプを避ける
- **医療アイコン：** 過度な医療感（注射・赤十字等）は避け、温かみのあるアイコンを使用

## 3. UX原則

### 重視する体験

1. **続けられること（継続性）**
   チェックインは1分以内に完了。通知・フィードバックで「また明日も使おう」と思わせる設計

2. **安心して使えること（プライバシー）**
   「あなたのデータは会社に見えません」を随所で再確認できるUI。相談画面は特に安心感を強調

3. **迷わないこと（シンプルさ）**
   機能が多くても、毎日使う「チェックイン→気づき→コンテンツ」の導線は一直線にする

### 避けたいUX

1. **「健康診断」感** - 数値・スコアを前面に出しすぎない。管理される感覚にならないように
2. **過剰な通知・催促** - 使ってない日に責めるようなプッシュ通知は送らない
3. **医療用語の多用** - 専門用語より平易な言葉で。「黄体期」より「月経前の時期」

## 5. インタラクション

### アニメーション方針
控えめに使う。チェックイン完了時・気づき表示時など、「完了した」「次へ進む」タイミングに限定。

### フィードバック
- ボタンクリック / タップ：即時の視覚フィードバック（色変化・ホバーエフェクト）
- 送信完了：「送信しました」トースト通知
- 相談返信：Web Push通知 / メール通知＋バッジ

### エラー表示
- やさしいトーンで「もう一度お試しください」
- 原因が明確な場合は具体的に（「ネットワークに接続されていません」等）
- 医療相談エラー時は「担当者に連絡します」と代替案を提示
